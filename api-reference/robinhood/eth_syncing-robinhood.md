# eth\_syncing - Robinhood

Returns sync status as an object if the node is currently syncing, or `false` if the node is fully synced and at chain tip.

## Parameters

* None

use reqwest::Client;
use serde_json::json;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = Client::new();

    let payload = json!({
        "jsonrpc": "2.0",
        "method": "eth_syncing",
        "params": [],
        "id": "getblock.io"
});

    let response = client
        .post("https://shared.us-east-1.getblock.io/<ACCESS-TOKEN>/")
        .header("Content-Type", "application/json")
        .json(&payload)
        .send()
        .await?;

    let result: serde_json::Value = response.json().await?;
    println!("{:#?}", result);

    Ok(())
}

## Response Parameters

| Field  | Type              | Description                                                                                       |
| ------ | ----------------- | ------------------------------------------------------------------------------------------------- |
| result | boolean \| object | `false` if fully synced; otherwise an object with `startingBlock`, `currentBlock`, `highestBlock` |

## Use Cases

* Routing reads only to fully-synced nodes
* Detecting nodes still in initial sync
* Monitoring sync progress

## Error Handling

| Status Code | Error Message     | Cause                                          |
| ----------- | ----------------- | ---------------------------------------------- |
| 403         | Forbidden         | Missing or invalid `<ACCESS-TOKEN>`            |
| -32602      | Invalid params    | Request parameters are missing or malformed    |
| -32603      | Internal error    | Server-side error while processing the request |
| 429         | Too Many Requests | Rate limit exceeded for your plan              |
